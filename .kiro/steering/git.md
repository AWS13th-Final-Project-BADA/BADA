# BADA — Git 컨벤션 (Git Steering)

> 커밋·브랜치·PR 작성 시 이 규칙을 따른다.

## 1. 커밋 메시지

- **한글로 작성한다.**
- 형식: `태그: 요약` (한 줄, 50자 이내 권장)
- 본문이 필요하면 빈 줄 후 상세 설명 추가 가능

### 태그 목록

| 태그 | 용도 |
|------|------|
| 기능 | 새로운 기능 추가 |
| 수정 | 버그 수정 |
| 리팩터 | 기능 변경 없는 코드 정리 |
| 문서 | 문서/주석만 변경 |
| 테스트 | 테스트 추가/수정 |
| 설정 | 설정 파일, 인프라, CI 변경 |

### 예시

```
기능: 다국어 번역 시스템 구현
수정: GPS 교차검증 시간대 비교 오류 해결
문서: steering 파일에 git 컨벤션 추가
테스트: 번역 배치 처리 단위테스트 추가
```

## 2. 브랜치 네이밍

- `feature/기능명` — 새 기능
- `fix/이슈명` — 버그 수정
- `docs/내용` — 문서 작업

## 3. PR

- 제목은 커밋 메시지와 동일하게 한글
- 본문에 "무엇을 했는지" + "테스트 방법" 간략히 기술
