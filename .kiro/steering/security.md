# BADA — 보안·개인정보 (Security Steering)

> BADA는 취약 외국인 노동자의 분쟁 증거(이름·임금·계좌·제3자 정보)를 다룬다. 민감도 최상.
> 코드·데이터 흐름·외부 호출을 생성할 때 이 규칙을 지킨다. "최소 수집 / 신뢰경계 / 원본 보존"이 원칙.

## 1. PII 마스킹 — Upstage 경로 한정 (가장 중요)

데이터가 **AWS 신뢰경계를 벗어나는 지점은 Upstage(외부 API) 하나뿐**이다. 거기에만 마스킹을 건다.

- **Bedrock Claude(Vision/Text)**: AWS 신뢰경계 내 + 학습 미사용 → 원문 전송 허용.
- **Upstage Document Parse**: AWS 밖 외부 API → **전송 전 정규식 마스킹 필수.**
  - 마스킹 대상: 계좌번호, 주민등록번호, 전화번호.
  - 방식: Python `re` 정규식 (MVP). Presidio/Comprehend 고도화는 Phase 2.
  - 마스킹은 "Upstage로 보낼 사본"에만 적용. **DB의 원본 OCR 텍스트는 무수정 보존** (대조 가능하게).

> ⚠️ 주의: "OpenAI로 직배송 안 함" 같은 문구를 쓰지 말 것. 스택에 OpenAI는 없다.
> 실제 외부 전송 지점은 Upstage이며, 방어 논리는 거기에만 적용한다.

## 2. 저장·암호화

- S3: **KMS 암호화** (원본 증거 + 생성 PDF).
- RDS: 전송구간 TLS, 저장 암호화.
- 비식별화 공유본(Phase 2): 이름·계좌·전화·제3자 정보 마스킹 후 생성.

## 3. 데이터 생명주기

- 일반 데이터(증거파일·PDF 등) 기본 보존: **90일**. 사용자 요청 시 **즉시 삭제**.
- **GPS 로그는 예외로 3년 보존** (임금채권 소멸시효 기준 — 근로기준법 제49조). seed/web_geo/app 데이터 모두 동일 정책 적용. 위치정보법상 "6개월 이상 보관" 요건은 이 기준으로도 충족된다.
- 최소 수집 원칙: 사건 정리에 필요한 자료만.
- GPS의 3년 안에서도 접근 빈도가 낮아지는 구간은 저장 등급만 낮춘다(S3 스토리지 클래스 전환). 삭제 시점 자체는 늦추지 않는다 — 상세는 `architecture.md` GPS 아카이빙 설계 참고.
- ⚠️ **미구현 상태**: `retention_days`(일반, 90일)·`gps_retention_days`(GPS, 3년) 둘 다 설정·환경변수로만 존재하고, 이를 읽어 기간 초과 데이터를 실제로 삭제하는 배치/로직은 아직 없다. 현재 사건 삭제(`DELETE /cases/{id}`)는 DB 레코드만 지우고 S3 원본은 그대로 남는다. MVP 데모 트래픽 규모에서는 보존기간 경과 사례가 발생하지 않아 우선순위를 낮췄으나, 실서비스 전환 시 반드시 구현해야 한다(개인정보보호법 "목적 달성 후 지체없이 파기" 요건).

## 4. 제3자 정보 (카톡 캡처)

- 카톡 캡처에는 사업주 등 제3자 개인정보가 섞인다 → PIPA상 쟁점.
- MVP: 증거로만 사용, 외부 공유본 생성 시 제3자 정보 마스킹(Phase 2). 약관에 수집·이용 범위 동의 포함.

## 5. 인증·접근

- Cognito 이메일 로그인. 사건/증거는 소유 사용자만 접근(행 수준 권한).
- Presigned URL은 짧은 만료시간. S3 버킷 퍼블릭 접근 차단.

## 6. 무결성 (선택, 어필 효과)

- Evidence Pack PDF 하단에 **SHA-256 해시 + 타임스탬프** 삽입 → 사후 위변조 입증.
- 구현은 백엔드 한 줄 수준(`hashlib`). GPS `is_mocked` 데이터는 증거에서 배제(domain.md 참조).

## 7. 카카오톡 증거 (법적 안전 범위 — 반드시 준수)

> 카톡 대화는 개인정보 처리에 해당할 수 있음. "본인 당사자 대화"만, 동의·마스킹·단기보관 전제로 처리.
> 상세 설계: `docs/ocr-roadmap.md` 5~6장.

- **본인이 참여한 대화만 업로드** 명시 + 업로드 전 **제출목적 처리 동의** 필수.
- 카톡 **원문 장기 저장 금지** → 필수 필드 + 원본 SHA-256 해시 우선 저장(원본성).
- 제3자 실명·전화·계좌·주민번호 **자동 마스킹**(security.pii). 분쟁 핵심 필드 low면 "확인 필요".
- 공개·공유 기능 제한(노동청 제출/본인 다운로드 범위). 원본/가공본 분리. 운영자 열람 최소화.
- 태깅 범위는 사건 관련(지급약속·체불인정·금액언급·근무지시)로 한정, 나머지는 가림.
- ⚠️ **출시 전 개인정보처리방침·동의문구·삭제정책·마스킹 기준 법률 전문가 검토 필수.**

### 업로드 동의 문구(초안)
> "본인이 직접 참여한 대화만 업로드하세요. 타인의 대화를 무단 업로드하지 마세요.
>  업로드 자료는 상담·신고 준비용 증거 정리 목적으로만 처리되며 민감정보는 자동 가림됩니다.
>  (본 도구는 법률자문이 아닙니다.)"

## 8. 약관·면책

- 이용약관에 "법률자문 아님" 동의 필수.
- 모든 산출물에 면책 고지 노출(product.md 4항).
