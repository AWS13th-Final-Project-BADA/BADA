# BADA — OCR 서비스화 개선 정리 (처음 → 현재)

> 담당 범위: 백엔드 + 프론트엔드 + OCR
> 핵심 설계 원칙(변하지 않음): **읽기·문장화·번역만 LLM, 계산·정렬·판정·대조는 규칙 코드.**
> 그래서 모든 금액·판정은 환각의 영향을 받지 않고, 근거(산식)를 댈 수 있습니다.

---

## 1. 한눈에 보기

| 구분 | 처음(초기 MVP 골격) | 현재 |
|------|------|------|
| OCR 엔진 | 사용자가 엔진 선택 | **Claude Vision 기본 고정**(정형문서만 Upstage 옵션) |
| 추출 필드 | 시급·금액·공제·날짜·발화 | + **연장/야간/휴일 시간·근무일수·계약기간·서명·4대보험 개별공제** |
| 정확성 검증 | 없음 | **sanity(내부 모순)** + **legal(법정 기준 미달)** + **교차대조** |
| 신뢰도 | LLM 자기보고(과신) | **근거 기반 confidence**(교차일치/sanity/단일출처) |
| 사용자 수정 | 시급 1개만 | **모든 추출 필드 인라인 편집 + 저장(HITL)** |
| 입력 | 파일 선택만 | **카메라 촬영 + 자동 압축·회전 보정·품질 경고** |
| 정확도 측정 | 없음(감으로) | **eval 골든셋 — 필드 단위 정확도 리포트** |
| 테스트 | 규칙 일부 | **워커 50 + 백엔드 10 자동 테스트** |

## 2. OCR 파이프라인

- **엔진 라우팅**: 정형문서(계약서·명세서·일정표)는 Upstage/Parseur 옵션, 비정형(카톡·사진·앱캡처)과 기본값은 **Claude Vision**. 사용자가 고를 필요 없음.
- **PDF 지원**: 이미지/PDF 자동 판별(document vs image 블록).
- **긴 응답 안정화**: max_tokens 상향 + 재시도 시 raw_text 축약으로 JSON 잘림 방지.
- **관대한 스키마**: `"1,795,680원"` 같은 문자열·null도 정수로 자동 변환해 추출 실패로 통째로 버려지지 않음.

## 3. 추출 품질 · 신뢰성

- **확장 필드**: 연장/야간/휴일 근로시간, 근무일수, 계약 시작·종료일, 서명 유무. 4대보험은 공제 개별 항목(국민연금/건강보험/장기요양/고용보험/소득세)으로.
- **sanity(타당성) 검사**: 통상임금<기본급, 실지급>지급총액, 지급계−공제계≠실지급 등 **숫자 모순**을 OCR 두 번 안 돌리고 잡음.
- **근거 기반 confidence**: LLM 자기보고 대신 **교차일치(다른 증거와 같은 값)·sanity 통과·단일출처**로 high/medium/low를 매김. 낮은 필드는 화면에서 노란색 ⚠로 강조.
- **HITL 편집**: 사용자가 모든 추출값을 직접 고쳐 "수정 저장" → 백엔드 저장 후 sanity·confidence 재계산. (손글씨 오독 등 보정)

## 4. 규칙 엔진 (판정·대조)

- **compare(검증포인트)**: 계약서↔명세서 시급, 명세서 실지급↔통장 입금 교차 대조 + 차액.
- **chat_evidence(증거력)**: 카톡 캡처를 상대식별·날짜·맥락·핵심문장·금액 5개 체크리스트로 채점 + 보강 안내.
- **guardrails(금지표현)**: "불법/체불 확정/무조건 받음" 같은 단정 표현을 중립 문구로 치환(오탐 방지 경계 정규식).
- **legal(법령 산식 매핑) — 신규**: 실제 노동법 기준을 상수·공식으로 박아 **법정 기준 미달**까지 점검.
  - 최저임금 미달(2026년 시급 10,320원 기준) + 부족액 산식
  - 연장·야간·휴일 **가산수당 추정**(가산분 0.5배)
  - 4대보험 **과다공제 의심**(표준 요율 대비 1.5배 초과만 보수적으로)
  - 원칙: 명백한 것만 자신있게, 애매하면 "전문가 확인".

## 5. 사용자 흐름 (UX)

- **촬영 흐름**: 카테고리별 📷 촬영 / 파일 선택. 업로드 전 EXIF 회전 보정·다운스케일·JPEG 압축, 해상도 낮음·캡처 너무 김 경고.
- **스텝퍼**: 올리기 → AI 추출 → 확인·수정 → 분석 단계 표시.
- **결과 화면**: 사건 요약·금액 비교·**증거 대조**·**⚖️ 법정 기준 점검**·공제·GPS·타임라인(확인필요·출처)·더 준비할 자료. 차이 없으면 헤드라인을 과장 없이 표시.
- **다국어**: 한국어/영어/베트남어 UI + 원문-번역 대조표.
- **제출용 리포트**: 위 내용을 인쇄/PDF용 HTML로(법정 기준 점검 섹션 포함).

## 6. 측정 · 품질 보증

- **eval/ocr_score.py(신규)**: 실제 샘플에 정답(gold)을 달아 **필드 단위 정확도**(시급·금액·공제·확장필드)를 숫자로 리포트. 이후 개선의 기준선.
- **자동 테스트**: 워커 50개(sanity·confidence·legal·compare·guardrails·schema 등), 백엔드 10개(업로드→OCR→수정저장→분석 E2E). `python -m pytest`로 실행.
