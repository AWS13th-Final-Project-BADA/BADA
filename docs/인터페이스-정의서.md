## 2. 컴포넌트 간 통신

### 2.1 모바일 앱 → Backend

| 프로토콜 | 인증 | 포맷 |
|---------|------|------|
| HTTPS (443) | Bearer JWT | JSON / Multipart |

### 2.2 Backend → SQS

| 방향 | 메시지 타입 | 트리거 |
|------|-----------|--------|
| 발행 | `{ type: "analyze_case", case_id }` | POST /analysis/{id} |
| 발행 | `{ task: "transcribe", evidence_id, s3_key, language_code }` | 음성 파일 업로드 시 |

### 2.3 Worker ← SQS

| 방향 | 방식 | 설정 |
|------|------|------|
| 수신 | Long-polling | wait=20초, max=5건/회 |
| 성공 | 메시지 삭제 | DeleteMessage |
| 실패 | 미삭제 → 재시도 → DLQ | maxReceiveCount 후 redrive |

### 2.4 Worker → 외부 서비스

| 대상 | 프로토콜 | 용도 |
|------|---------|------|
| AWS Bedrock | HTTPS (SDK) | LLM 요약/문장화 (Claude Sonnet 4) |
| AWS Transcribe | HTTPS (SDK) | 음성→텍스트 |
| AWS Translate | HTTPS (SDK) | 텍스트 번역 |
| Upstage API | HTTPS (REST) | OCR (Document AI) |
| S3 | HTTPS (SDK) | 증거 파일 읽기, PDF 저장 |
| RDS | TCP (5432) | 결과 저장 |

### 2.5 Backend → 외부 서비스

| 대상 | 용도 |
|------|------|
| S3 | 파일 업로드/다운로드, Presigned URL 생성 |
| SQS | 분석/전사 작업 발행 |
| RDS PostgreSQL | 영구 데이터 저장 |
| Google/Kakao/Naver OAuth | 소셜 로그인 code→token 교환 |

### 2.6 Prometheus → Backend

| 경로 | 간격 | 방식 |
|------|------|------|
| `GET /metrics` | 15초 | HTTP scrape |

## 3. 데이터 흐름

### 3.1 증거 업로드 → 분석 → PDF

```
모바일 앱
  │
  │ POST /cases/{id}/evidences/upload (multipart)
  ▼
Backend
  ├─ 파일 → S3 evidence 버킷 (SSE-KMS 암호화)
  ├─ Evidence 레코드 → RDS
  ├─ (음성) SQS → { task: "transcribe", evidence_id, s3_key }
  │
  │ POST /cases/{id}/extract
  ├─ 각 Evidence → OCR → extracted_entities (JSON) → RDS
  │
  │ POST /analysis/{case_id}
  └─ SQS → { type: "analyze_case", case_id }
         │
         ▼
      Worker
         ├─ RDS에서 Case + Evidence 읽기
         ├─ 규칙엔진 계산 (미지급/공제/GPS/교차대조/법정기준)
         ├─ LLM 타임라인 + 요약 생성
         ├─ AnalysisResult + TimelineEvent + TranslationPair → RDS
         ├─ Case.status = "completed" → RDS
         └─ PDF 생성 → S3 report 버킷
                │
                ▼
모바일 앱
  │ GET /analysis/{case_id}         → 결과 JSON
  │ GET /analysis/{case_id}/report/pdf → PDF 다운로드
```

### 3.2 인증 플로우

```
모바일 앱
  │ WebBrowser.openAuthSessionAsync
  │ GET /auth/google/login?redirect_uri=bada://auth
  ▼
Backend → 302 Redirect → Google OAuth authorize
  │
  │ (사용자 로그인 완료)
  ▼
Google → 302 → GET /auth/google/callback?code=xxx&state=yyy
  │
Backend
  ├─ code → token 교환 (Google API)
  ├─ token → userinfo (email, name)
  ├─ DB: User upsert (provider=google, provider_id=xxx)
  ├─ JWT 발급 (sub=user.id, exp=7일)
  └─ 302 Redirect → bada://auth?token={jwt}
         │
         ▼
모바일 앱 → SecureStore에 JWT 저장 → 이후 모든 요청에 Bearer 헤더
```

### 3.3 AI 채팅 플로우

```
모바일 앱
  │ POST /chat/messages { message, case_id, language }
  ▼
Backend (ai_chat_orchestrator)
  ├─ classify_intent(message) → 의도 분류
  ├─ classify_legal_risk(message) → safe/caution/blocked
  ├─ (blocked) → 법적 판단 불가 폴백 응답 반환
  ├─ retrieve_rag_context(db, question, intent) → RagChunk[]
  ├─ generate_llm_chat_answer(Bedrock Claude) → answer
  ├─ apply_output_guardrail(answer) → 법률 단정 차단
  ├─ annotate_terms_for_language(answer) → 용어 주석
  └─ { answer, intent, risk_level, sources, next_actions, disclaimer }
```

### 3.4 GPS 핑 무결성

```
모바일 앱
  │ POST /gps/{case_id}/ping { lat, lng, ts }
  ▼
Backend
  ├─ prev_hash = DB에서 마지막 GpsLog의 chain_hash 조회
  ├─ new_hash = SHA256(prev_hash + lat + lng + ts)
  ├─ distance = Haversine(ping좌표, 직장좌표)
  ├─ status = distance ≤ radius ? "IN_WORKPLACE" : "OUTSIDE"
  ├─ GpsLog 저장 (lat, lng, ts, chain_hash, status, is_mocked, is_delayed)
  └─ 응답: { status, distance_m, chain_hash }
```

## 4. 외부 서비스 연동 상세

| 서비스 | 인증 방식 | 사용 위치 | 설정 키 |
|--------|----------|----------|---------|
| AWS S3 | ECS Task Role (IAM) | Backend, Worker | `S3_BUCKET` |
| AWS SQS | ECS Task Role (IAM) | Backend(발행), Worker(수신) | `SQS_QUEUE_URL` |
| AWS Bedrock | ECS Task Role (IAM) | Worker, Backend(chat) | `BEDROCK_MODEL_ID` |
| AWS Transcribe | ECS Task Role (IAM) | Worker | — |
| AWS Translate | ECS Task Role (IAM) | Worker | — |
| AWS KMS | ECS Task Role (IAM) | S3 연동 자동 | `KMS_KEY_ID` |
| Upstage API | API Key (Header) | Worker OCR | `UPSTAGE_API_KEY` |
| Google OAuth | Client ID/Secret | Backend auth | `GOOGLE_CLIENT_ID/SECRET` |
| Kakao OAuth + 채널 | REST API Key/Secret | Backend auth + 챗봇 | `KAKAO_REST_API_KEY/SECRET` |
| Naver OAuth | Client ID/Secret | Backend auth | `NAVER_CLIENT_ID/SECRET` |

## 5. DB 스키마 (주요 테이블)

| 테이블 | PK | 주요 컬럼 | 관계 |
|--------|-----|-----------|------|
| User | id (UUID) | email, name, provider, provider_id, preferred_lang | 1:N → Case |
| Case | id (UUID) | user_id, workplace_name, employer_name, agreed_hourly_wage, status | 1:N → Evidence, AnalysisResult |
| Evidence | id (UUID) | case_id, category, s3_key, ocr_text, ocr_status, extracted_entities | N:1 → Case |
| AnalysisResult | id (UUID) | case_id, suspected_unpaid, deduction_items, timeline_summary, pdf_s3_key | N:1 → Case |
| TimelineEvent | id (UUID) | case_id, event_type, title, description, event_date, confidence | N:1 → Case |
| TranslationPair | id (UUID) | case_id, source_text, translated_text, evidence_type | N:1 → Case |
| Workplace | id (UUID) | case_id, center_lat, center_lng, radius_m | 1:1 → Case |
| GpsLog | id (UUID) | case_id, lat, lng, ts, chain_hash, status, is_mocked, is_delayed_upload | N:1 → Case |
| CommunityPost | id (UUID) | author_id, title, body, category, language, status | 1:N → Comment |
| CommunityComment | id (UUID) | post_id, author_id, body, status | N:1 → Post |
| RagDocument | id (UUID) | title, source_org, category | 1:N → RagChunk |
| RagChunk | id (UUID) | document_id, content, embedding, section | N:1 → Document |
| KakaoLink | id (UUID) | user_id, kakao_user_id | 1:1 → User |
