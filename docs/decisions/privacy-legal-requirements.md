# BADA 프로젝트 개인정보보호 법적 조항 적용 가이드

이 문서는 카카오모빌리티의 개인정보처리방침을 기준으로, **BADA 프로젝트(노동 분쟁 증거 수집 앱)**의 아키텍처와 기능에 필수적으로 적용되어야 하는 법적 조항과 실제 구현 방안을 정리한 가이드입니다.

---

## 1. 제1조: 개인정보의 수집 목적, 항목 및 보유기간
BADA 앱은 사용자의 민감한 노동 정보와 갤러리 사진을 다루므로 수집 원칙을 엄격히 지켜야 합니다.

- **적용 대상**: 구글 로그인 정보(이메일, 이름), 사용자가 업로드한 캡처본(급여명세서, 근로계약서 등에 포함된 성명, 연락처, 주소, 급여액 등)
- **법적 요구사항 (개인정보보호법 제15조)**:
  - 서비스 제공에 필요한 최소한의 정보만 수집해야 합니다.
  - 목적(노동 분쟁 증거 수집 및 분석)이 달성되거나, 회원이 탈퇴하면 파기해야 합니다.
- **BADA 구현 과제**:
  - 회원가입 시 "수집 항목, 목적, 보유기간"을 명시한 동의창(UI) 필수 구현.
  - 사용자가 업로드하는 사진 속에 제3자(사장님, 동료)의 정보가 포함될 수 있으므로, 이에 대한 법적 면책 조항이나 비식별화(가명처리) 안내 필요.

## 2. 제2조 & 제4조: 개인정보의 처리 위탁 및 국외 이전
BADA는 자체 서버가 아닌 클라우드와 외부 AI API를 강력하게 의존하는 서비스입니다.

- **적용 대상**: AWS (EC2, RDS, S3, Transcribe, Bedrock), Anthropic (Claude 3.5 Sonnet)
- **법적 요구사항 (개인정보보호법 제26조, 제28조의8)**:
  - 외부 서버나 AI 모델에 개인정보를 넘겨 처리하게 하므로 **'처리 위탁'**에 해당합니다.
  - 특히 Anthropic Claude API나 AWS 리전이 해외에 있을 경우 **'국외 이전'**에 해당하므로 사전 동의가 필수입니다.
- **BADA 구현 과제**:
  - 처리방침에 수탁자(Amazon Web Services, Anthropic)와 위탁 목적(클라우드 인프라 제공, AI 텍스트 분석)을 명시.
  - "사용자의 증거 데이터가 AI 분석을 위해 해외 서버로 전송될 수 있음"을 명시하고 국외 이전 동의 확보.

## 3. 제6조: 개인위치정보의 처리
BADA는 출퇴근 기록을 위해 지오펜싱(GPS)을 사용하므로 위치정보법의 적용을 받습니다.

- **적용 대상**: `gps.js` 및 백그라운드 지오로케이션으로 수집되는 위도/경도 핑 데이터
- **법적 요구사항 (위치정보법)**:
  - 위치기반서비스 사업자 신고 대상일 수 있습니다.
  - 위치정보 수집 시 반드시 별도의 동의를 받아야 합니다.
  - **위치정보 수집/이용/제공 사실 확인자료를 시스템에 자동 기록하고 6개월 이상 보관해야 합니다.**
- **BADA 구현 과제**:
  - 현재 `gps.js`에서 `/cases/{id}/gps/ping`으로 수집하는 로그는 DB에 **3년 보관**(security.md 기준, 위치정보법 "6개월 이상" 요건을 상회) 되도록 유지 (현재 구조 부합 — 단, 자동 삭제 로직은 아직 미구현, 아래 4항 참고).
  - 앱에서 위치 권한(Foreground/Background)을 요청할 때 명확한 수집 목적(출퇴근 증거 수집) 고지.
  - 사건이 종료되거나 사용자가 위치 기록을 삭제하면 즉시 DB에서 복구 불가능하게 파기.

## 4. 제9조: 개인정보의 파기
법정 의무 보관 기간과 서비스 파기 원칙을 시스템적으로 구현해야 합니다.

- **적용 대상**: 사용자 계정, RDS에 저장된 사건/증거 메타데이터, S3에 저장된 사진/음성 원본
- **법적 요구사항**: 목적 달성 후 지체 없이 파기 (종이문서는 파쇄, 전자파일은 복구 불가능하게 영구 삭제).
- **BADA 구현 과제**:
  - 앱 내에 **'회원 탈퇴' 및 '사건 삭제' 버튼** 필수 구현.
  - 삭제 버튼 클릭 시, RDS DB 레코드 삭제뿐만 아니라 연동된 **AWS S3 버킷의 원본 이미지/녹음 파일도 API를 통해 즉시 삭제(`s3:DeleteObject`)**되도록 백엔드 로직 구현.
  - ⚠️ **현재 미구현**: `DELETE /cases/{id}`, `DELETE /evidences/{eid}`는 DB 레코드만 삭제하고 S3 원본은 그대로 남는다. `retention_days`(일반 데이터, 90일)·`gps_retention_days`(GPS, 3년) 값도 설정에만 존재하고 이를 근거로 실행되는 자동 파기 배치가 없다. MVP 데모 규모에서는 우선순위를 낮췄으나 실서비스 전환 전 필수 구현 항목.

## 5. 제13조: 안전성 확보조치 (보안)
민감한 임금, 계약 정보가 있으므로 해킹/유출 방지 조치가 필수입니다.

- **법적 요구사항**: 비밀번호 등 주요 정보 암호화, 네트워크 구간 암호화, 접근 통제.
- **BADA 구현 과제**:
  - (현재 운영 인프라 TODO 적용) **RDS 스토리지 암호화** 활성화 (KMS 사용).
  - ALB(로드밸런서)에 **ACM(SSL/TLS 인증서)**을 부착하여 모든 통신을 `HTTPS`로 강제.
  - S3 Evidence 버킷의 Public Access 완전 차단 및 KMS 암호화 유지 (현재 적용됨).
