> 문서 버전: 1.0
> 최종 갱신: 2026-06-26
> 작성 기준: 구현된 코드베이스 역공학 기반 (현행 시스템 반영)

---

## 1. 개요

### 1.1 프로젝트 목적

BADA는 **외국인 근로자의 임금체불 증거를 체계적으로 수집·분석·패키징**하여, 노동부 진정 또는 법률 상담 시 제출할 수 있는 "증거 패키지(Evidence Pack)"를 자동 생성하는 서비스이다.

### 1.2 대상 사용자

| 사용자 | 설명 |
|--------|------|
| 외국인 근로자 | 임금체불을 겪고 있는 비한국어 화자. 한국어 문서 이해 어려움 |
| 상담 지원자 | 근로자를 돕는 NGO/상담소 직원 |
| (간접) 노동부/법원 | 최종 증거 패키지 수신자 |

### 1.3 지원 언어

- UI: 한국어(ko), 영어(en), 베트남어(vi), 캄보디아어(km), 네팔어(ne), 인도네시아어(id), 태국어(th), 일본어(ja)
- 증거 분석 결과: 원문(ko) + 사용자 모국어 번역 대조표

### 1.4 시스템 구성

- 모바일 앱 (React Native + Expo)
- Backend API (Python FastAPI)
- Worker (SQS 비동기 처리)
- 인프라 (AWS, Terraform IaC)
- 모니터링 (Prometheus + Grafana + CloudWatch)

### FR-01. 사용자 인증

| ID | 요구사항 | 우선순위 | 상태 |
|----|---------|---------|------|
| FR-01-01 | 소셜 로그인(Google, Kakao, Naver) OAuth 2.0 지원 | P0 | 구현됨 |
| FR-01-02 | JWT 기반 세션 관리 (발급, 검증, 만료 7일) | P0 | 구현됨 |
| FR-01-03 | 모바일 앱 딥링크(`bada://auth#token=`) 콜백 | P0 | 구현됨 |
| FR-01-04 | Cognito Hosted UI 로그인 (레거시, 전환 예정) | P2 | 구현됨 (비활성 전환 중) |
| FR-01-05 | 데모 모드 — 인증 없이 단일 유저 자동 로그인 | P2 | 구현됨 (개발 전용) |
| FR-01-06 | OAuth state 파라미터로 open redirect 방지 | P0 | 구현됨 |

**동작 설명**:
1. 앱에서 `GET /auth/{provider}/login?redirect_uri=bada://auth` 호출
2. Backend가 해당 provider의 authorize URL로 리다이렉트
3. 사용자가 소셜 로그인 완료 → provider가 `GET /auth/{provider}/callback?code=...` 콜백
4. Backend: code→token 교환 → userinfo 조회 → DB User upsert → JWT 발급
5. `bada://auth?token={jwt}`로 앱 딥링크 리다이렉트 → 앱이 SecureStore에 토큰 저장

### 4.1 외부 서비스 의존성

| 서비스 | 용도 | 설정 모드 |
|--------|------|-----------|
| AWS S3 | 증거 파일 / PDF 저장 | `storage_mode=s3` |
| AWS SQS | 비동기 작업 큐 (분석/전사) | `sqs_queue_url` 설정 시 |
| AWS Transcribe | 음성→텍스트 변환 (STT) | `transcription_dispatch_mode=sqs` |
| AWS Bedrock (Claude) | LLM 답변 생성, 요약, 문장화 | `ai_chat_mode=bedrock` |
| AWS Translate | 텍스트 번역 | Worker 내 자동 선택 |
| AWS KMS | 증거 암호화 키 관리 | `kms_key_id` |
| AWS Cognito | 레거시 인증 (전환 예정) | `auth_mode=cognito` |
| Upstage Document AI | OCR (이미지→구조화 텍스트) | `UPSTAGE_API_KEY` |
| Google OAuth | 소셜 로그인 | `google_client_id/secret` |
| Kakao OAuth + 채널 API | 소셜 로그인 + 챗봇 | `kakao_rest_api_key/secret` |
| Naver OAuth | 소셜 로그인 | `naver_client_id/secret` |

### 4.2 모드별 동작

| 설정 | 값 | 동작 |
|------|---|------|
| `auth_mode` | demo | 인증 없이 데모 유저 자동 사용 |
| | oauth | 소셜 OAuth JWT 필수 |
| | cognito | Cognito 토큰 검증 |
| `storage_mode` | local | ./uploads 로컬 저장 |
| | s3 | AWS S3 버킷 저장 |
| `ai_chat_mode` | mock | 규칙 기반 Mock 응답 |
| | bedrock | Bedrock Claude 실호출 |
| `provider_mode` | local | 로컬 더미 provider |
| | aws | 실제 AWS 서비스 호출 |

## 6. 용어 정의

| 용어 | 설명 |
|------|------|
| Evidence Pack | 증거 패키지 — 분석 결과 + 타임라인 + 번역 대조표를 포함한 PDF |
| Case | 사건 — 한 근로자의 한 사업장에 대한 임금체불 건 |
| Evidence | 증거 — 업로드된 파일 1건 (이미지/PDF/음성/텍스트) |
| OCR | 광학 문자 인식 — 이미지에서 텍스트 추출 |
| STT | Speech-to-Text — 음성에서 텍스트 추출 |
| Entity | 엔티티 — OCR 결과에서 추출한 구조화된 정보 (금액, 날짜, 시급 등) |
| Geofence | 지오펜스 — GPS 좌표가 설정된 반경 내인지 판정 |
| Hash Chain | 해시 체인 — GPS 핑의 순서/무결성을 보장하는 연쇄 해시 |
| DLQ | Dead Letter Queue — 처리 실패 메시지를 격리하는 대기열 |
| Guardrail | 가드레일 — LLM 출력의 법률 단정/환각을 차단하는 안전장치 |
| RAG | Retrieval-Augmented Generation — 문서 검색 기반 답변 생성 |
