## 5. Infra (`infra/`)

> ⚠️ 리소스는 도메인별 `.tf` 파일로 분리되어 있다(`main.tf` 단일 파일에 몰려 있지 않음).
> `main.tf`는 locals(name_prefix·이미지 해석), data 소스(AZ/AMI/Route53 zone), 공통 태그만
> 남은 **스텁**이며 대부분의 리소스는 아래 전용 파일로 이동됐다. (총 `.tf` 15개)

| 파일 | 역할 | 주요 리소스 |
|------|------|-----------|
| `main.tf` | 공통 정의(스텁) | locals(name_prefix·image), data(AZ/AMI/Route53 zone), common_tags |
| `network.tf` | 네트워크 | VPC, Public/Private Subnet, IGW, Route Table, **NAT Gateway(단일, egress)**, **S3 Gateway Endpoint**, SG(alb/ecs/rds/ssm), ALB, Target Group, ACM, HTTPS 리스너·호스트 라우팅, Route 53 |
| `compute.tf` | 컴퓨트/컨테이너 | ECR(backend/worker/frontend†), ECS Cluster(Container Insights), Capacity Providers(FARGATE + FARGATE_SPOT), Task Definition·Service(backend/worker/frontend†), X-Ray 사이드카, SSM bastion EC2 |
| `data.tf` | 데이터/스토리지/큐 | KMS, S3(evidence/report/alb-logs) + Lifecycle, SQS + DLQ, RDS(primary Single-AZ + rehearsal Multi-AZ), Cognito(pool/client/domain/Google IdP)‡, Secrets Manager, SSM 파라미터 |
| `autoscaling.tf` | 오토스케일링 | Backend CPU 70% + Worker SQS backlog-per-task Target Tracking (min=1/max=3) |
| `waf.tf` | WAF | WAFv2 REGIONAL Web ACL(IP평판 + Common + KnownBadInputs) + ALB 연결 + CloudWatch 로깅 |
| `security.tf` | 보안 모니터링 | GuardDuty Detector, Security Hub(FSBP), GuardDuty HIGH→EventBridge→SNS |
| `observability.tf` | 관측성 | CloudWatch Log Group(backend/worker/xray/frontend), SNS + 이메일 구독, Alarm 14개 |
| `iam.tf` | IAM | ECS Task Execution Role, Backend/Worker 최소권한 Task Role, SSM DB 접근 Role |
| `monitoring.tf` | 모니터링 스택 | Prometheus/Grafana ECS, EFS, Cloud Map, ALB 라우팅 (`monitoring_enabled` 게이트) |
| `github-actions.tf` | CI/CD 권한 | OIDC Provider, PR plan-role(읽기전용), deploy-role(`develop` 한정) |
| `mcp-readonly.tf` | 개발 도구 | CloudWatch/Logs 읽기 전용 AssumeRole |
| `variables.tf` | 입력 변수 | 리소스 토글 다수(`frontend_enabled`, `*_autoscaling_enabled`, `worker_fargate_spot_enabled`, `security_monitoring_enabled`, `use_rehearsal_multiaz_db_as_app_db` 등) |
| `outputs.tf` | 출력값 | ALB DNS, ECR URL, RDS endpoint |
| `providers.tf` | Provider 설정 | AWS provider, 리전, `default_tags`(Project/Environment/ManagedBy) |

> † Frontend 리소스(ECR/Task/Service/Target Group/리스너 규칙)는 `count = var.frontend_enabled ? 1 : 0`로
> 게이트되어 있으며 기본값 `false`라 실제 생성되지 않는다(코드는 존재, 배포 안 됨).
> ‡ Cognito 리소스는 무조건 생성되지만 앱은 소셜 OAuth를 쓰므로 미사용 레거시다(2절 참고).
