# OCR 개선 로드맵 — 증거 세트 → 비교 가능한 구조 데이터

> 방향: "문서 1장 OCR"이 아니라 **근로계약서·급여명세서·통장 입금내역·출근부를 증거 세트로 묶어
> 비교 가능한 필드셋으로 정규화**한다. 핵심은 입력 정규화 → 구조 추출 → 사건 연결.
> 담당: OCR(=백엔드+프론트+OCR). 안전 원칙은 `.kiro/steering/security.md` 준수.

## 1. 파이프라인 (4단계)
1. **문서 분류** — 업로드 시 contract/statement/payment/schedule/chat 자동·수동 분류
2. **구조 추출** — 문서 타입별 JSON 스키마로 필드 추출 + confidence
3. **정규화** — 날짜(ISO), 금액(콤마/원 제거 정수), 시간(시:분), 통화 통일
4. **검증 포인트 생성** — 비교 키 산출(계약 시급↔명세서 시급, 명세서 실지급↔통장 입금, 출근부 시간↔수당 근거)

## 2. 문서별 추출 필드 스키마

### 근로계약서 (contract)
| 필드 | 설명 |
| --- | --- |
| workplace_name, employer_name, worker_name | 사업장/사업주/근로자 |
| work_start_date, work_end_date | 입사일/종료일 |
| hourly_wage / monthly_wage | 시급/월급 |
| contracted_hours | 소정근로시간(주/일) |
| pay_day | 지급일(매월 N일) |
| deduction_clauses[] | 공제 동의 조항(기숙사비 등) |

### 급여명세서 (statement)
| 필드 | 설명 |
| --- | --- |
| pay_month, pay_date | 지급월/지급일 |
| gross_pay (지급총액), base_pay (기본급) | |
| allowances[] (수당: 연장·야간·휴일·기타) | line items |
| deductions[] (공제: 소득세·4대보험·기타) | line items |
| total_deduction (공제총액), net_pay (실지급액) | |
| overtime_hours, night_hours, holiday_hours | 시간 근거 |

### 통장 입금내역 (payment)
| 필드 | 설명 |
| --- | --- |
| transactions[] | 거래 line items |
| ↳ date, amount, payer_name(입금자), memo(적요) | |
| recurring_pattern | 반복 입금 패턴(월급성 여부) |

### 출근부/스케줄 (schedule)
| 필드 | 설명 |
| --- | --- |
| days[] | 날짜별 line items |
| ↳ date, clock_in, clock_out, total_hours, break_min, is_night | |
| period_total_hours | 기간 총 근무시간 |

> 공통: 모든 필드에 `confidence(high/medium/low)`. 핵심 분쟁 필드(이름·금액·날짜)는 low면 **확인 필요** 표기(HITL).

## 3. 정규화 규칙
- 날짜 → `YYYY-MM-DD` (2026.6.2 / 26/06/02 / 6월 2일 → 통일)
- 금액 → 정수(원), 콤마·"원"·공백 제거. (예: "186,300원" → 186300)
- 시간 → `HH:MM` 24h, 총시간은 소수 시간(8.5)
- 통화 → KRW 기본

## 4. 검증(비교) 키 — `worker/rules/compare.py` (구현됨)
| 키 | 비교 | 판정 |
| --- | --- | --- |
| hourly_wage | 계약 시급 ↔ 명세서 시급 | match/mismatch/missing |
| net_vs_deposit | 명세서 실지급액 ↔ 통장 입금액 | 차액 계산 |
| hours_vs_allowance | 출근부 총시간 ↔ 명세서 수당 근거 | (2차) |
- 결과는 "확정"이 아니라 "차이 확인 필요"로만 표기(product.md 표현 정책).

## 5. 카카오톡 증거 모듈 (체불 입증용 대화 증거 정리)
> 단순 OCR이 아니라 **원본성·연속성·상대 특정성**을 보강해 "캡처 텍스트"를 "증거 기록"으로.

### 기능
- **대화 유형 분류**: 지급약속 / 체불인정 / 금액협의 / 근무지시 / 퇴사후정산 / 사과·지연통보 (이미 utterance.kind로 분류 중)
- **연속 캡처 연결**: 여러 장 → 시간순 복원, 앞뒤 맥락 누락 검사
- **메타데이터**: 날짜·시각·발화자·캡처 순서·파일 생성일
- **원본성 보강**: 원본 이미지 **SHA-256 해시** + 업로드 시각 + 파일명 보관(변조 추적)
- **대화 내보내기 병합**: 카톡 "대화 내용 내보내기" txt와 캡처 OCR 상호 대조 → 신빙성↑

### 증거 이벤트 단위(event unit) 예시
```
{ "date":"2026-05-02", "speaker":"사업주",
  "text":"이번 주 금요일까지 40만원 줄게", "kind":"wage_promise",
  "amount":400000, "source_evidence":"screenshot_03", "confidence":"high" }
```

### 신뢰 점수 규칙 (가산)
| 항목 | 점수 |
| --- | --- |
| 원본 이미지 있음 | +1 |
| 연속 캡처 있음 | +1 |
| 날짜/시각 보임 | +1 |
| 상대 식별 가능 | +1 |
| 내보내기 txt 있음 | +2 |
→ 합산해 high/medium/low 표시. 점수 낮으면 누락 경고.

### 누락 경고
"날짜 안 보임", "상대 식별 불가", "중간 대화 누락 가능", "핵심 문장 앞뒤 맥락 부족"

## 6. 법적 안전장치 (필수, security.md 연동)
- **본인 당사자 대화만 업로드** 명시 + 업로드 전 **제출목적 처리 동의**
- 카톡 **원문 장기 저장 X** → 필수 필드 + 원본 해시 우선 저장
- 제3자 실명·전화·계좌·주민번호 **자동 마스킹**
- 공개·공유 기능 제한(노동청 제출/본인 다운로드 범위)
- 보관기간 제한(사건 종료 후/단기 자동 삭제), 원본/가공본 분리, 운영자 열람 최소화
- ⚠️ **출시 전 법률 전문가 검토 필수** (개인정보처리방침·동의문구·삭제정책·마스킹 기준)

### 업로드 동의 문구(초안)
> "본인이 직접 참여한 대화만 업로드하세요. 타인의 대화를 무단으로 업로드하지 마세요.
>  업로드한 자료는 상담·신고 준비를 돕기 위한 증거 정리 목적으로만 처리되며,
>  민감정보는 자동 가림 처리됩니다. (본 도구는 법률자문이 아닙니다.)"

## 7. 구현 우선순위 (MVP: 명세서+입금내역+계약서 3종 정확도 우선)
1. ✅ **비교(검증) 엔진** `rules/compare.py` — 시급·실지급↔입금 (이번 구현)
2. 문서 타입별 추출 스키마/프롬프트 (line items: 수당·공제·거래·일자)
3. 정규화 레이어(날짜·금액·시간)
4. 카톡 원본성 해시 + 신뢰점수 + 마스킹 + 동의 UI
5. 출근부 + 연속 캡처 병합(2차)
