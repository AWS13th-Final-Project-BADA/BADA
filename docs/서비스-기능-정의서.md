# BADA 서비스 기능 정의서

> 최종 갱신: 2026-06-26
> "이 서비스가 무엇을 하는가"를 기능 단위로 정의한 문서.

---

## 1. 서비스 개요

외국인 근로자의 **임금체불 증거를 수집·분석·패키징**하여, 노동부 진정 또는 법률 상담 시 제출할 수 있는 Evidence Pack(PDF)을 자동 생성하는 서비스.

**핵심 가치**: 한국어를 못 읽는 외국인 근로자가 급여명세서·계약서·카톡 등을 사진으로 찍어 올리면, 시스템이 자동으로 "얼마를 못 받았는지" 분석하고, 모국어 번역과 함께 증거 리포트를 만들어준다.

---

## 2. 주요 기능

### F1. 사용자 인증

| 항목 | 설명 |
|------|------|
| 방식 | 소셜 OAuth (Google / Kakao / Naver) |
| 토큰 | JWT (7일 만료), 모바일 SecureStore 저장 |
| 딥링크 | `bada://auth?token={jwt}` — 앱으로 자동 복귀 |
| 보안 | state nonce, redirect_uri 화이트리스트 |
| 자동 로그아웃 | API 401 응답 시 저장 토큰 삭제 후 로그인 화면 이동 (서버 세션 없는 stateless JWT) |

### F2. 사건(Case) 관리

사건 = 한 근로자의 한 사업장에 대한 임금체불 건.

| 기능 | 설명 |
|------|------|
| 생성 | 사업장명, 사업주명, 약속 시급, 근무기간, 문제유형 입력 |
| 목록 | 본인 사건만 최신순 조회 |
| 상세 | 사건 정보 + 연결된 증거/분석 현황 |

### F3. 증거 수집

증거 = 임금체불을 입증하는 자료 1건.

| 기능 | 설명 |
|------|------|
| 파일 업로드 | 이미지/PDF/음성 → S3 암호화 저장 |
| 자동 분류 | 6개 카테고리: 계약서, 근무일지, 급여명세서, 카톡대화, 통장내역, 기타 |
| 빠른 스캔 | 여러 장 동시에 카테고리만 분류 (OCR 없이, 빠름) |
| 정밀 평가 | 1장 분류 + OCR + 키워드 교차검증 (정확함) |
| Presigned 업로드 | 대용량 파일 직접 S3 업로드 (모바일 최적화) |
| 수동 입력 | 텍스트 직접 입력 (구두 약속 등) |

### F4. OCR / STT 추출

| 기능 | 설명 |
|------|------|
| 이미지 OCR | 급여명세서 등에서 금액·날짜·시급·공제 자동 추출 |
| 음성 STT | 녹음 파일 → AWS Transcribe → 텍스트 변환 |
| 엔티티 구조화 | 추출 결과를 JSON 스키마로 강제 (Pydantic 검증) |
| 수동 보정 | 사용자가 OCR 결과를 수동으로 수정 가능 |

**추출 대상**: 날짜, 금액(시급/월급/지급액), 근무시간, 공제항목, 사업장명, 대화 발화

### F5. 임금체불 분석

사건의 증거들을 종합하여 "얼마를 못 받았는지" 계산하고, 법적 기준과 비교.

| 단계 | 내용 |
|------|------|
| 1. 미지급 산정 | 시급 × 근무시간 - 입금액 = 미지급 추정액 |
| 2. 공제 분류 | 4대보험 / 식비 / 기숙사 / 기타로 자동 분류 |
| 3. 법정 기준 검사 | 최저임금 미달, 가산수당 누락, 보험 과다공제 |
| 4. 증거 교차대조 | 계약서 시급 ↔ 명세서 지급 ↔ 통장 입금 비교 |
| 5. GPS 출근 검증 | 지오펜스 내 출퇴근 + 카톡 시간 대조 |
| 6. 누락 증거 식별 | 필수 카테고리 대비 미보유 항목 표시 |
| 7. 타임라인 생성 | 사건 이벤트 시계열 + LLM 문장화 |
| 8. 번역 대조표 | 원문(한국어) ↔ 모국어 번역 쌍 |
| 9. 요약 | LLM 기반 전체 요약 (가드레일 적용) |

**안전 장치**:
- 숫자 환각 감지 (소스에 없는 금액 → 폴백)
- 법률 단정 표현 차단 ("~해야 합니다" → "~할 수 있습니다")
- 규칙엔진 결과는 LLM 실패 시에도 항상 반환

### F6. Evidence Pack (보고서/PDF)

| 기능 | 설명 |
|------|------|
| HTML 보고서 | 임금체불 정밀 분석 리포트 (웹 뷰) |
| PDF 생성 | 인쇄/제출용 Evidence Pack (**항상 한국어** — 노동부/법원 제출용) |
| 다운로드 | S3에서 직접 다운로드 |
| 화면 다국어 | 분석 결과 화면은 사용자 언어로 실시간 번역 (Amazon Translate) |

**PDF 내용**: 사건 요약, 미지급 금액, 공제 분석, 타임라인, 번역 대조표, 증거 목록
**다국어 설계**: DB 저장=한국어, 화면 조회=사용자 locale로 실시간 번역, PDF=한국어 고정

### F7. GPS 근무 증거

"실제로 출근했다"를 GPS로 입증하는 기능.

| 기능 | 설명 |
|------|------|
| 직장 등록 | 직장 위치(좌표) + 인정 반경(m) 설정 |
| 핑 기록 | 앱이 주기적으로 GPS 좌표 전송 |
| 지오펜스 | 반경 내 → IN_WORKPLACE, 밖 → OUTSIDE |
| 위변조 방지 | 해시 체인 (이전 해시 + 현재 데이터 = SHA256) |
| 신뢰도 필터 | Mocked GPS / Delayed Upload 자동 배제 |
| 일별 요약 | 날짜별 출근 인정 횟수 집계 |

### F8. AI 상담 채팅

노동법 관련 질문에 AI가 답변. **법률 자문이 아닌 정보 제공**.

| 기능 | 설명 |
|------|------|
| 의도 분류 | 일반상담 / 증거질문 / 절차안내 등 |
| 법적 리스크 차단 | "이기냐/지냐" 류 판단 질문 → 안전 폴백 |
| RAG 검색 | 노동법 문서 벡터 DB에서 근거 검색 |
| LLM 답변 | Bedrock Claude로 자연어 답변 생성 |
| 가드레일 | 법률 단정 차단 + 면책 조항 자동 첨부 |
| 사건 연동 | case_id 전달 시 해당 사건 맥락 반영 |
| 용어 주석 | 전문용어에 괄호 설명 자동 추가 |

### F9. 카카오 챗봇

카카오톡 채널을 통한 간편 접근 (앱 미설치 사용자 대응).

| 기능 | 설명 |
|------|------|
| 계정 연동 | 앱에서 발급한 6자리 코드를 카톡에 입력 → 링크 |
| 체불 진단 | "시급 만원" 입력 → 최저임금 대비 차액 계산 |
| 상태 조회 | 현재 사건의 분석 진행 상태 표시 |
| 체크리스트 | 보유/미보유 증거 카테고리 시각화 |
| 누락 안내 | 아직 없는 증거 유형 + 수집 가이드 |
| 다국어 | ko/en/vi/zh 언어별 카드 응답 |

### F10. 커뮤니티

외국인 근로자 간 정보 공유 공간.

| 기능 | 설명 |
|------|------|
| 게시글 | 작성/조회/수정/삭제, 카테고리·언어·인기순 필터 |
| 댓글 | 작성/수정/삭제 |
| 좋아요/저장 | 토글 방식 |
| 번역 | 게시글/댓글을 지정 언어로 번역 |
| 신고 | 부적절 콘텐츠 신고 + 관리자 처리 |
| 안전 검사 | 작성 전 유해 콘텐츠 필터링 |

---

## 3. 핵심 워크플로

```
사용자 여정 (Happy Path):

[로그인] → [사건 생성] → [증거 업로드 (사진 찍기)]
    → [OCR 추출] → [분석 요청]
    → [Worker: 미지급 계산 + 법정 기준 + 교차대조 + 타임라인 + 요약]
    → [PDF 생성]
    → [결과 확인 + PDF 다운로드]
    → 노동부/상담소에 제출
```
